Features

Jug is a Symfony console application that turns Twig templates into straight HTML. On top of that, it does the following:

Twig powered

Since the templates are built in Twig, you get access to blocks, includes and macros. Build reusable snippets and use them everywhere throughout your project.

You also get access to all the default filters.

Multi-lang ready

Jug includes the symfony/translations package, which means the |trans filter is available.

If multiple locales are configured, you'll get a version of your site in each locale.

Cache friendly

Jug adds a build-specific hash to all your assets for easy cache-busting on deploy. This means you can aggressively cache those static assets in your webserver config for top-notch performance.

Optimized images

Install optipng and jpegoptim on your local system and both will be used on all images when you build your site.

Dead simple PHP configuration

Manage config, settings and other variables in config/jug.php. The content of the entire array will be passed to all your Twig templates.


<?php

return [
    'default_locale' => 'en',
    'locales' => [
        'en',
    ],
    'hash' => bin2hex(random_bytes(4)),
    'year' => (new DateTime('now'))->format('Y')
];

Markdown

Use the markdown Twig filter to parse content as markdown:

'some **markdown** content'|markdown

Or use the Twig apply block to transform entire blocks:

{% apply markdown %} # More markdown! Wow! * a list * more lists {% endapply %}

Watch command for easy development

Running ./vendor/bin/jug watch will watch the files in the source folder and build your site every time you make a change.

You can find more in-depth examples on the demo page.
